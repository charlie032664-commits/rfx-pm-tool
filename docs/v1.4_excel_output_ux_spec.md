# v1.4 Excel Output UX Polish Spec — PM Internal Review Version

> **Status:** spec / design only. **No export code is changed by this document.**
> Implementation is deferred to a future PR (see §7).

## 1. Purpose

This spec defines the **PM internal review** Excel output style for the v1.4+
Compliance Matrix (`compliance_matrix.xlsx`). The current workbook (reviewed via
`compliance_matrix (79).xlsx`) is functional, but the PM-facing internal review
sheet needs clearer column naming, plain-language (Traditional Chinese) field
explanations, and a more professional, less "warning-colored" layout. This
document is the agreed target for a later code PR against
`scripts/export_excel.py`.

## 2. Scope

**In scope**

- The **Compliance Matrix** main sheet (and the per-category sheets that share its
  column layout).
- The **PM internal review** version only.
- Column **naming** (display labels).
- Column **ordering**.
- **Header comments / Chinese field notes**.
- A proposed **Field Guide** sheet.
- **Basic style rules** (fills, fonts, borders, freeze, widths, alignment).
- **AI internal field positioning** (move to the right, visually separated).

**Out of scope (for now)**

- SOP / Sequence fields — explicitly **excluded** per user decision.
- A separate **customer-facing** workbook or sheet.
- LLM / normalization **behavior** changes.
- Requirement **extraction** logic.
- Response **merge** logic (`_merge_responses` stays keyed by `req_id`).

### 3.2 Mapping notes

- **PM Reviewed Requirement** = the **final** requirement text used for PM review
  and response (PM edit → AI-normalized → original fallback). This is the
  response baseline.
- **Customer Requirement (Original)** = the **original customer wording** when

  traceability only**.
- **AI Rewrite Status**, **AI Confidence**, **PM AI Review Status** = internal AI
  **audit** fields.

  with the PM response flow (cols 6, 9–12).

## 5. Style Proposal

### 5.1 Current actual styling (verified in `scripts/export_excel.py`)

Re-checked the real fills/fonts before recommending changes — several common
complaints don't match the code:

- **Header:** fill `1F4E79` (dark navy blue) + **bold white** text, font **Arial**.
  → Already professional; **keep** (only consider the font swap in §5.2).
- **Body font:** Arial (Req ID cells explicitly `Arial 10`; other cells inherit default).
- **Borders:** thin **light gray** `D9D9D9`. → Already correct; keep.
- **Freeze:** main sheets freeze top row (`A2`); summary sheet freezes `C2`.
- **Req ID is NOT bright green.** Actual: `RFQ-*` → light-green fill `EAF3DE` +
  **dark green text** `27500A`; `AI-*` → light-blue fill `E6F1FB` + **dark blue
  text** `0C447C`. It is already a subtle light-fill/dark-text scheme.
- **Conditional fills currently in use:**
  - Priority `MUST` row → `F8CBAD` (light salmon).
  - Status: NEED_REVIEW `FFF2CC`, NEW `E3F2FD`, INTERNAL_ALIGN `E8EAF6`,
    ASK_CUSTOMER `FFF3E0`, READY_FOR_RESPONSE `EAF3DE`, CLOSED `F5F5F5`,
    AUTO_SKIP `E7E6E6`.
  - Normalized column: REVIEW → `FFE699` (yellow); has-content & not-REVIEW →
    `DDF0E0` (light green); Rewrite Review cell = REVIEW → `FFD966` (bold yellow).

  → The **yellow** the user noticed is the **normalization/REVIEW** highlighting
  (`FFE699` / `FFD966`), which sits on the AI columns that we are moving to the
  right block. Once moved, the yellow stops appearing in the middle of the PM flow.

### 5.2 Recommended style (professional, not over-styled)

- **Header:** keep dark blue fill + white bold text. Optional: keep navy but
  ensure it's the single professional blue across all sheets.
- **Font:** Calibri or Aptos, **10–11 pt** body, **11 pt bold** header. (Migrating
  from Arial → Calibri/Aptos is optional polish, low priority.)
- **Req ID:** drop the green-text treatment for `RFQ-`; prefer **dark gray
  (`404040`)** or **dark blue (`1F4E79`)** text for *both* prefixes for a calmer,
  consistent look. Keep a very light fill (or none) — avoid bright colors.
- **Editable PM fields** (Responsible Team, Stakeholder, Compliance Status, PM
  Reviewed Requirement, Our Response, Gap/Notes, Evidence): subtle fill **only if
  needed** (e.g. very light blue `F2F7FC`), or no fill. **Do not** make the whole
  sheet look like a warning area.
- **AI internal fields** (cols 14–17): light **gray** (`F3F3F3`) or **blue-gray**
  (`ECEFF3`) background to visually separate them from PM-facing columns. Reserve
  yellow strictly for genuine "needs review" flags within `PM AI Review Status`.
- **Borders:** thin light-gray `D9D9D9` (keep current).
- **Alignment:** wrap text for requirement/response columns (PM Reviewed
  Requirement, Customer Requirement, Our Response, Gap/Notes, Evidence, AI Parsed
  Requirement); **vertical top** alignment for all data rows.
- **Freeze:** freeze top row; **consider freezing the first column** (Req ID) so
  the row identity stays visible when scrolling right into the AI block.
- **Width:** wider columns for requirement/response text; compact columns for
  Priority / Compliance Status / Responsible Team / AI Confidence.

**Traditional Chinese and English**, rather than cramming all explanations into
header comments. Proposed layout:

| Column | 中文名稱 | English | 說明 (Chinese) | Description (English) | Editable | Internal only |
|--------|---------|---------|----------------|-----------------------|:---:|:---:|

- One row per column from §3.1, reusing the Chinese text from §4.
- Place the sheet near the front (e.g. right after `By_Category_Summary`) so PMs
  see it early; or at the end as a reference appendix — see Open Questions.
- This keeps the main sheet visually clean while still onboarding new reviewers.

## 7. Implementation Plan for Later Code PR

> **Do not implement now.** This is the plan for a future PR; this document is
> spec-only.

Likely the only file changed: **`scripts/export_excel.py`**.

Future implementation steps:

1. **Centralize column definitions** in one list/dict (e.g. a `COLUMNS` structure
   holding `key`, `label`, `chinese`, `editable`, `internal`, `width`, `wrap`),
   replacing the flat `HEADERS` list.
2. **Add display labels** (the renames in §3.3).
3. **Add Chinese comments** on header cells (§4).
4. **Reorder columns** to the §3.1 order.
5. **Move AI fields to the right** (cols 14–17) and apply the AI block fill.
6. **Apply professional styles** (§5.2) — header, Req ID text color, editable/AI
   fills, freeze first column.
7. **Add the `Field Guide` sheet** (§6).
8. **Validate** the generated workbook opens and **keeps the 12-sheet structure**
   (Compliance Matrix, By_Category_Summary, six category sheets, Glossary, Notes,
   Skipped, Excluded) — plus the new Field Guide if added (then 13 sheets;
   confirm count expectation in the PR).
9. **Run export-only validation** on one safe case first (e.g. `20260129_IBM_RFQ`),
   writing to a gitignored temp folder.
10. **Do not run LLM or Full Pipeline** — this is an export-formatting change only.
