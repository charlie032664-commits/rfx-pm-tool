## v1.2 — Robust background pipeline runner (merged)

- **Detached background worker** (`scripts/pipeline_worker.py`): runs the 4-stage
  pipeline as a standalone process that survives Streamlit refresh/rerun.
- **Persistent job status** (`scripts/job_status.py`): `runs/<case>/job_status.json`
  + `jobs.jsonl` with stage/status/provider/model/pid/timestamps (secret-masked
  errors).
- **Pipeline lock protection**: the worker honors the same `.pipeline.lock` as the
  synchronous run, so background and sync runs are mutually exclusive.
- **Provider status** (Streamlit sidebar): provider/model + per-var presence +
  ready, presence-only (no secrets).
- **Safe LLM smoke test** (`scripts/llm_smoke_test.py`): tiny generation +
  synthetic extraction; no customer data.

## v1.3 — Runtime evaluation + cache foundation (merged)

- **Runtime estimator + large-run guard** (`scripts/runtime_estimator.py` + UI
  expander): estimates files / chunks / runtime by provider; risk thresholds
  >1h warning, >4h strong warning.
- **Model output comparison** (`scripts/compare_model_outputs.py`): stats-only
  comparison of two run dirs (counts, dup ids, missing/empty fields,
  category/status distributions, matrix rows) — no LLM, no customer text.
- **Guarded cache-reuse design** (`docs/cache_reuse_design.md`): documents
  current report-only cache, correctness risks, and a guarded plan (default OFF,
  `--use-extraction-cache`, signature must add `doc_schema_hash`). True reuse NOT
  implemented yet.
- **AtlasRFQ comparison baseline** (`docs/v1.3_atlasrfq_comparison.md`): zero-drift
  reference for future cache work.

## Known limitations

- Background launch button validated end-to-end but the *interactive* "refresh
  mid-run" click-path is confirmed only via successful detached completions.
- Cache is **report-only** — no LLM work is skipped/reused yet.
- Internal `qwen3-coder-next` is slow (~20s/call); large LLM-heavy extractions
  (e.g. SilverPeak ~3085 chunks) are ~17h on internal — use OpenAI or run
  overnight with explicit intent.
- `run_history.jsonl` is written by the UI path only (CLI/worker runs use
  `job_status.json`).

## Recommended usage

- Configure provider in `.env`; launch via `run_rfx.bat`; verify with
  `python scripts/env_loader.py`.
- For long runs, use **"Run Full Pipeline in background (beta)"** so a refresh
  can't abandon the run; watch the "Pipeline job status (persistent)" panel.
- Click **"Estimate runtime"** before a large run; heed the >4h strong warning.
- Avoid re-running Extract unnecessarily — use **Enrich + Format + Export** when
  `requirements.json` already exists.
