# v1.4 No-AI Flow Test Mode

## Why this mode exists

PM workflow testing needs a fast way to exercise upload, review, edit, and matrix-generation screens without waiting for an LLM extraction run. The no-AI mock extraction CLI creates a small requirements JSON from local case files so UI flow changes can be tested quickly and repeatedly.

## What it does

`scripts/mock_extract_requirements.py` scans text-like files under an inbound case folder and emits requirement-like candidate lines. It uses simple local rules only:

- requirement keywords such as `shall`, `must`, `required`, `support`, `comply`, `requirement`, and `should`
- simple numbered, lettered, and bullet list patterns

The output includes mock IDs such as `MOCK-001`, `MOCK-002`, plus metadata marking the run as `analysis_mode: no_ai_mock`.

## What it does not do

This mode does not call an LLM, interpret RFQ intent, parse Office/PDF content, classify requirements deeply, assess compliance risk, or produce final extraction quality. It is intentionally shallow and deterministic.

## Why it is safe

The CLI reads local files from the case folder, writes one JSON output file, and prints only counts plus the output path. It does not dump source document contents to the console. It does not call external APIs, paid services, internal LLM endpoints, or production services.

## Why output must not be used as final RFQ extraction

The candidates are heuristic matches, not validated requirements. Keyword and list-pattern matching can miss important requirements, include irrelevant lines, duplicate concepts, and assign placeholder metadata. The output exists only to test UI flow and must not be used for customer-facing RFQ analysis or final compliance matrices.

## Later UI connection

The intended next step is to let a UI flow use this JSON as temporary mock extraction input for Review/Edit and then Generate Matrix testing. That future connection should keep the mode clearly labeled as no-AI mock data and should prevent users from confusing it with final extraction results.

## Validation command examples

```bash
python -m py_compile scripts/mock_extract_requirements.py
python scripts/mock_extract_requirements.py --help
python scripts/mock_extract_requirements.py --case-dir inbound/sample_case --out runs/sample_case/requirements.json --limit 20
```
