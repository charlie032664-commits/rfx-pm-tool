## Problem statement

Responding to an RFQ/RFP is slow, manual, and error-prone. A single RFQ arrives
as a pile of inconsistent Word, Excel, and PDF files containing **100–200+
requirements**. Today a Product Manager must:

- read every document and hand-extract requirements,
- normalize wildly inconsistent wording,
- categorize and prioritize each item,
- chase owners for compliance responses, and
- assemble a clean, customer-ready **compliance matrix** by hand.

This is **half a day to a full day of effort per RFQ**, it doesn't scale, and
copy-paste mistakes create real commercial and compliance risk.

---

## Solution statement

An **AI engine** that turns raw RFQ documents into a customer-ready compliance
matrix in minutes, while keeping the PM in control of everything that ships. The
AI does the extraction, cleaning, and assembly; the PM reviews and approves. The
final matrix can be regenerated **instantly and deterministically** — no LLM
re-run — every time the PM edits a requirement or a response.

---

## Agent / AI Engine framing

The product is structured as a pipeline of **specialized agents**, surfaced today
as a guided, numbered workflow with a human checkpoint. The same decomposition is
what makes it ready to become a more autonomous **agent platform**.

```
Source Files ─▶ [Source File Agent] ─▶ [Extraction Agent] ─▶ [Normalization Agent]
                                                                      │
                                                                      ▼
   Compliance Matrix ◀─ [Matrix Generation Agent] ◀─ [Response Agent] ◀─ [PM Review Agent]
                                                                ▲                │
                                                                └── human-in-the-loop ──┘
```

### Workflow agents

| Agent | Responsibility | Maps to |
|-------|----------------|---------|
| **Source File Agent** | Ingest and organize heterogeneous RFQ files per case | Step 1 |
| **Requirement Extraction Agent** | Pull structured requirements from messy documents | Step 3 |
| **Requirement Normalization Agent** | Clean and standardize requirement language | Advanced option after Step 3, before Step 4 (*Improve requirement wording with AI*) |
| **PM Review Agent** | Support PM editing, recategorizing, and deleting rows | Step 4 |
| **Compliance Response Agent** | Capture and persist per-requirement compliance answers | Step 5 |
| **Matrix Generation Agent** | Assemble the customer-ready compliance matrix (deterministic, no LLM) | Step 6 |

A **Pre-check** gate (Step 2) and **Download** delivery (Step 7) bracket the agents.

---

## Business value

- **Time:** RFQ-to-matrix drops from ~half/full day to **minutes** of active PM time.
- **Scale:** handles 100–200+ requirements per RFQ without manual copy-paste.
- **Quality:** consistent categorization, normalized wording, fewer transcription errors.
- **Governance:** nothing reaches the customer without explicit PM approval.
- **Iteration cost:** regenerating the matrix after edits is **instant and free**
  (pure export — no LLM tokens, no pipeline re-run).
- **Reusability:** the agent decomposition generalizes beyond RFx.

### Before / after PM effort

| Dimension | Before (manual) | After (v1.4) |
|-----------|-----------------|--------------|
| Extract requirements | Hours of reading + typing | AI engine, minutes |
| Normalize wording | Manual, inconsistent | AI-normalized, consistent |
| Categorize / prioritize | Manual | AI-proposed, PM-adjusted |
| Build matrix | Hand-assembled Excel | One-click generate |
| Regenerate after a change | Re-edit the spreadsheet by hand | One click, instant |
| Customer accuracy control | Hope nothing was mistyped | PM-approved, deleted rows excluded |

## Demo scenario

Using a prepared case (see `docs/v1.4_demo_script.md`):

1. Show the numbered Step 1 → Step 7 PM workflow.
2. In **Step 4**, edit a requirement and delete a junk row live — the governance moment.
3. In **Step 5**, fill a compliance response.
4. In **Step 6**, click **Generate Compliance Matrix**; highlight the
   *"Using reviewed requirements."* banner and the instant, no-LLM regenerate.
5. In **Step 7**, download and open the customer-ready `compliance_matrix.xlsx`.

Key talking point: *"AI proposes, the PM disposes, and the matrix only ever
contains what the PM approved."*

## Current MVP architecture

- **Frontend / orchestration:** Streamlit app (`app.py`) presenting the
  Step 1 → Step 7 guided workflow.
- **AI engine:** extraction + normalization scripts (LLM-backed) producing
  structured requirements.
- **Human-in-the-loop store:** PM review saved as `requirements_reviewed.json`
  (`schema_version = v1.4_review_mvp`); responses saved as
  `responses/<case>/responses.json`, keyed by `req_id`.
- **Deterministic export:** `scripts/export_excel.py` builds the multi-sheet
  `compliance_matrix.xlsx` from a **priority-selected** requirement source
  (reviewed → clean → mock → raw) merged with saved responses. The export does
  **not** call an LLM or run the pipeline.
- **Per-case data layout:** `runs/<case>/` for inputs/outputs,
  `responses/<case>/` for compliance answers.

**Reviewed-schema guarantees (validated):** `id → req_id`, `deleted: true` rows
skipped, `pm_comment` omitted from the customer matrix, responses merged by `req_id`.

## Future Open Cloud / Agent platform direction

- **Promote each step to a deployable agent** with its own interface and tests,
  callable independently or as a chain.
- **Increase autonomy gradually** — let high-confidence agents (extraction,
  normalization) run unattended while review stays a human gate, with confidence
  thresholds deciding what needs PM eyes.
- **Open Cloud hosting** for multi-user, multi-tenant RFQ processing with
  per-case isolation and audit trails.
- **Agent memory & learning** — reuse past PM edits/decisions to improve
  extraction and normalization over time.
- **Connectors** — pull RFQs directly from email/SharePoint and push matrices
  back to CRM/deal rooms.

## Risks / safeguards

| Risk | Safeguard |
|------|-----------|
| AI extracts a wrong/garbage requirement | **Step 4 PM review**; deleted rows never reach the matrix |
| Internal notes leak to customer | `pm_comment` is **omitted** from the customer-facing matrix |
| Stale or wrong source used for export | Explicit **source-priority** selection + UI banner showing which source is in use |
| Accidental cost/latency on regenerate | Step 6 is a **deterministic export** — no LLM, no pipeline |
| Lost responses on regenerate | Responses persist per case and **merge by `req_id`** on every export |
| Demo failure from a live AI run | Demo uses a **pre-analyzed case**; live extraction is optional follow-up |

## Suggested judging highlights

- **Agentic decomposition that's real, not slideware** — each agent maps to a
  working step in a shipped tool.
- **Governed automation** — a hard human-in-the-loop checkpoint; the AI never
  ships unreviewed content.
- **Deterministic, zero-cost regeneration** — instant matrix refresh with no LLM
  tokens, a strong efficiency and trust story.
- **Tangible business value** — half/full-day of manual PM work compressed to
  minutes, at RFQ scale.
- **Clear runway** — a credible path from MVP workflow to an autonomous,
  cloud-hosted agent platform.
- **Evidence, not claims** — an overnight export regression
  (`docs/v1.4_overnight_export_validation_report.md`) scanned **19** case folders
  and ran the matrix export on every case with a valid source: **8 PASS / 0 FAIL**,
  exercising the reviewed, clean, and mock source paths — **no LLM, no Full
  Pipeline**. Validated guarantees: Step 6 source priority
  (reviewed → clean → mock → raw), responses merge by `req_id`, `deleted: true`
  reviewed rows skipped, and `pm_comment` omitted from the customer-facing matrix.
