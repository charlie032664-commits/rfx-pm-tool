## Completed features
1. **Runtime estimator + large-run guard** — `runtime_estimator.py` estimates
   file count, text size, chunk count, and runtime using provider-aware
   per-call seconds (internal `qwen3-coder-next` ≈ 20s/call; OpenAI ≈ 0.6s/call).
   Risk thresholds: >1h warning, >4h strong warning. UI expander shows
   provider/model/files/chunks/runtime/risk with the appropriate warning.
   Verified: **SilverPeak on internal ≈ 17h 8m → HIGH (strong warning)**; on
   OpenAI ≈ 31m → OK; AtlasRFQ ≈ 21m → OK.
2. **Model output comparison** — `compare_model_outputs.py --left --right --out`
   compares two run dirs: requirement/glossary/notes counts, duplicate ids,
   missing/empty fields, category/must-level/status distributions, compliance
   matrix row count, output-file availability. Markdown report; statistics only,
   no LLM, no customer text. Verified on two sandbox AtlasRFQ runs (28 vs 28).
3. **Guarded cache-reuse design** — `docs/cache_reuse_design.md` documents
   current report-only behavior, what must change for true reuse, correctness
   risks (text/prompt/model/chunking/schema/stale-ids), and a guarded plan
   (default OFF, `--use-extraction-cache`, signature must add a `doc_schema_hash`,
   clean-vs-reuse comparison before shipping). True reuse intentionally NOT
   implemented (not yet low-risk).
