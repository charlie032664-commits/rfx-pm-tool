# v1.4 — Comparison Report UI (Design)

Goal: a Streamlit panel wrapping `scripts/compare_model_outputs.py` so a PM can
compare two run folders without the CLI. Stats only — no LLM, no customer text.

## Existing pieces

- `scripts/compare_model_outputs.py` — `collect_stats(run_dir)` + `build_report(left, right)`;
  CLI: `--left --right --out`. Reports counts, duplicate ids, missing/empty
  fields, category/must-level/status distributions, compliance matrix rows, file
  availability.
- `docs/v1.3_atlasrfq_comparison.md` — example output (28 vs 28, zero drift).

## UI panel design

- Location: a new "Compare runs" expander (Step 3 area, or a small dedicated
  section). Read-only.
- **Left / right pickers:** two `st.selectbox`s listing available run dirs that
  contain `requirements_clean.json` (discover via
  `runs/**/requirements_clean.json`). Default left = current case's `runs/<case>`.
- **Run comparison:** a button calls `compare_model_outputs.collect_stats` on each
  side and `build_report(left, right)`; render the returned markdown inline with
  `st.markdown`. Also show the headline metrics as `st.metric` columns
  (requirements L/R, matrix rows L/R, duplicate-id counts, drift = "none" when all
  equal).
- **Save report:** optional "Save report" button → writes
  `runs/<caseL>_vs_<caseR>_compare.md` (gitignored). Never auto-commit.

## Guardrails

- **No LLM**, **no pipeline** — `compare_model_outputs` only reads existing JSON/
  XLSX.
- **No customer text** — the script already emits counts/distributions only; the
  UI must not add a raw-requirements dump.
- Reads are cheap; if needed wrap `collect_stats` in `st.cache_data(ttl=…)` keyed
  by run dir + clean.json mtime.

## Nice-to-have (later)
- A "drift summary" badge: PASS (all equal) / DIFF (highlight which metrics
  differ) computed by diffing the two `collect_stats` dicts.
- CSV export of the distribution tables (simple, no new deps).

## Risk
- Low. Pure read/format wrapper around an existing tested script. Implement in a
  small additive PR; the only care needed is the run-dir discovery + selectbox
  state.
