# v1.4-rc1 Manual Demo Checklist

## Purpose

A step-by-step checklist for **manually validating and demoing** the v1.4-rc1 PM
workflow **without running the LLM or the Full Pipeline**. Use it to confirm the
release candidate behaves correctly in a live UI and to drive a safe, repeatable
demo for internal users or AI innovation competition judges. Every action here is
either read-only or a deterministic, no-LLM operation (review edits, response
saves, the Step 6 export).

## Current RC Baseline

- **Tag:** `v1.4-rc1`
- **Target commit:** `99e61a1`
- **Final PM workflow:**
  1. Step 1: Case & Source Files
  2. Step 2: Pre-check
  3. Step 3: Analyze RFQ
  4. Advanced option: Improve requirement wording with AI *(collapsed, after Step 3 / before Step 4)*
  5. Step 4: Review Requirements
  6. Step 5: Fill Compliance Responses
  7. Step 6: Generate Compliance Matrix
  8. Step 7: Download Results

## Setup

- [ ] Confirm you are on `v1.4-rc1` or master `99e61a1`
      (`git checkout v1.4-rc1` for a clean RC demo).
- [ ] Confirm `.env` is configured (but do **not** run any live LLM/extraction during the demo).
- [ ] Launch Streamlit from the project directory:
      `streamlit run app.py`
- [ ] Open the local browser URL shown in the terminal.

## Pick a Safe Pre-analyzed Case

Use a case that already has analysis output (`requirements_clean.json` or
`requirements_reviewed.json`) plus saved responses. Recommended:

- [ ] `20260129_IBM_RFQ`
- [ ] `2018_SilverPeak_EC-XS`
- [ ] `2022_10_12_AtlasRFQ`

> **Do not start a fresh analysis during the demo** — extraction can call the LLM
> and take time. Always demo on a pre-analyzed case.

## Demo Checklist

- [ ] **Step 1 — Case & Source Files:** select a pre-analyzed case.
- [ ] **Step 2 — Pre-check:** confirm the pre-check section renders (input/validation status).
- [ ] **Step 3 — Analyze RFQ:** confirm the section appears and shows existing
      analysis output — **do not run it**.
- [ ] **Advanced option: Improve requirement wording with AI:** confirm it appears
      **after Step 3 and before Step 4**, **collapsed by default**. Safe to expand
      and describe — **do not run it**.
- [ ] **Step 4 — Review Requirements:** confirm the reviewed-requirements table
      appears. Safe to edit / recategorize / delete a row and **save** (no LLM).
- [ ] **Step 5 — Fill Compliance Responses:** confirm the section appears. Safe to
      fill a status + vendor comment and **save** (saved per case).
- [ ] **Step 6 — Generate Compliance Matrix:** confirm the **Generate** button
      appears and the **requirement-source message** shows which source is in use
      (*"Using reviewed requirements." / "Using cleaned requirements." / "Using
      fallback requirement source: `<filename>`."*). Safe to click **Generate**
      (export-only, no LLM); confirm the success message.
- [ ] **Step 7 — Download Results:** confirm the Download section appears.
      **Download and open `compliance_matrix.xlsx`** to show the deliverable.

## Demo Guardrails

**Do not click:**

- ❌ Analyze RFQ / Full Pipeline
- ❌ Run everything
- ❌ Normalize / *Improve requirement wording with AI*
- ❌ Any LLM action
- ❌ `.env` or model settings

**Safe to click:**

- ✅ Step 4 review / edit / save
- ✅ Step 5 response / save
- ✅ Step 6 Generate Matrix
- ✅ Step 7 Download Results

## Expected Demo Story

AI extracts the requirements from the raw RFQ, the **PM reviews and approves**
them (editing wording, deleting junk), owners **fill compliance responses**, and
the **compliance matrix is generated deterministically** — merging the latest
reviewed requirements with saved responses, **without rerunning the LLM or the
pipeline**. AI proposes, the PM disposes, and the matrix only ever contains what
the PM approved.

## Pass Criteria

- [ ] UI order is correct (Step 1 → Step 7).
- [ ] The AI cleanup option is in the right location (after Step 3, before Step 4, collapsed).
- [ ] Reviewed requirement changes can be saved.
- [ ] Compliance responses can be saved.
- [ ] Generate Matrix completes successfully.
- [ ] The downloaded Excel opens.
- [ ] No LLM / Full Pipeline is triggered during the demo.
